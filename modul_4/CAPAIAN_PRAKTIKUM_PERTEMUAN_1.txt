Capaian Praktikum Pertemuan 1
==============================

Mahasiswa dapat memahami dan mampu mengimplementasikan aplikasi chat multi-client berbasis TCP socket dengan sistem broadcast messaging, threading architecture untuk multiple client handling, serta manajemen koneksi client menggunakan list management. Mahasiswa juga dapat memahami konsep chat room atau group chat dimana satu server dapat menangani multiple client secara bersamaan, berbeda dengan komunikasi point-to-point pada modul sebelumnya.

Indikator Capaian
-----------------

Mahasiswa dapat menguji atau menjalankan program sesuai contoh pada modul dan dapat menyelesaikan masalah pada soal yang diberikan. Mahasiswa mampu memahami perbedaan antara komunikasi socket point-to-point (1:1) dan multi-client (1:N), serta dapat mengimplementasikan threading architecture untuk menangani multiple client connection secara paralel. Mahasiswa juga dapat memahami konsep broadcast messaging dimana pesan dari satu client dikirim ke semua client yang terhubung, sistem nickname dengan protokol NICK untuk identifikasi client, serta manajemen koneksi client menggunakan list management untuk tracking semua client yang sedang terhubung. Mahasiswa juga mampu memahami implementasi join dan leave notifications untuk memberikan informasi real-time tentang status client dalam chat room.

Landasan Teori
--------------

Pemrograman Berbasis Jaringan mempelajari pembuatan aplikasi yang saling berkomunikasi melalui jaringan menggunakan protokol seperti TCP (Transmission Control Protocol). TCP merupakan protokol transport connection-oriented yang memungkinkan pengiriman data dengan jaminan keandalan, urutan data, dan deteksi error. Protokol ini memerlukan proses handshake tiga arah (three-way handshake) untuk membentuk koneksi terlebih dahulu sebelum pengiriman data, sehingga lebih reliable dibandingkan UDP meskipun memiliki overhead yang lebih besar. TCP banyak digunakan pada aplikasi yang menuntut keandalan data, seperti aplikasi chat, web browsing, email, dan file transfer.

Dalam praktiknya, komunikasi TCP dilakukan menggunakan socket antara client dan server. Socket TCP menggunakan SOCK_STREAM yang menjamin pengiriman data secara berurutan dan andal. Server socket akan mendengarkan (listen) pada port tertentu dan menunggu koneksi dari client. Setelah koneksi terbentuk, server dapat menerima (accept) koneksi dari client. Berbeda dengan komunikasi point-to-point yang hanya mendukung satu client dalam satu waktu, aplikasi multi-client memungkinkan server untuk menerima multiple koneksi dari berbagai client secara bersamaan.

Aplikasi chat multi-client berbasis TCP socket memungkinkan komunikasi group chat atau chat room dimana satu server dapat menangani multiple client secara bersamaan. Untuk mengimplementasikan aplikasi multi-client, diperlukan teknik threading dimana setiap koneksi client diproses dalam thread terpisah. Threading architecture memungkinkan server untuk melayani banyak client secara paralel tanpa blocking, sehingga ketika satu client mengirim pesan, server dapat tetap menerima koneksi dan pesan dari client lainnya.

Sistem broadcast messaging merupakan komponen penting dalam aplikasi chat multi-client. Ketika satu client mengirim pesan, server akan menerima pesan tersebut dan mem-broadcast ke semua client yang terhubung, termasuk client yang mengirim pesan tersebut. Implementasi broadcast biasanya dilakukan dengan mengiterasi semua client dalam list clients dan mengirim pesan yang sama ke setiap client. Ini menciptakan efek chat room dimana semua peserta dapat melihat semua pesan yang dikirim oleh peserta lainnya.

Manajemen koneksi client dilakukan dengan menggunakan list management. Server menyimpan semua socket client dalam list clients dan semua username dalam list nicknames. Ketika client baru terhubung, server menambahkan socket client dan username ke dalam list yang sesuai. Ketika client disconnect, server menghapus socket client dan username dari list, kemudian mem-broadcast notifikasi bahwa client tersebut telah keluar dari chat room.

Sistem nickname atau username digunakan untuk identifikasi client dalam chat room. Protokol NICK digunakan untuk meminta username dari client ketika pertama kali terhubung. Server mengirim pesan 'NICK' ke client baru, dan client merespons dengan mengirim username yang telah diinput user. Server kemudian menyimpan username tersebut dan menggunakannya untuk identifikasi dalam pesan yang di-broadcast.

Join dan leave notifications memberikan informasi real-time tentang status client dalam chat room. Ketika client baru bergabung, server mem-broadcast pesan "{username} Bergabung!" ke semua client yang sudah terhubung, memberikan informasi bahwa ada client baru yang masuk. Ketika client disconnect, server mendeteksi error pada koneksi dan mem-broadcast pesan "{username} keluar!" ke semua client yang masih terhubung, memberikan informasi bahwa client tersebut telah meninggalkan chat room.

Dual threading pada client memungkinkan komunikasi non-blocking. Client menggunakan dua thread terpisah yaitu receive thread untuk menerima pesan dari server dan write thread untuk mengirim pesan ke server. Receive thread terus berjalan dalam loop untuk menerima pesan dari server, sementara write thread menunggu input dari user dan mengirim pesan ke server. Ini memungkinkan client untuk terus menerima pesan sambil menunggu input dari user, menciptakan pengalaman chat yang lebih interaktif dan real-time.

Fitur-fitur yang dapat diimplementasikan dalam aplikasi chat multi-client socket meliputi multi-client support untuk menangani multiple client secara bersamaan, threading architecture untuk setiap client connection, sistem nickname dengan protokol NICK untuk identifikasi, broadcast messaging untuk mengirim pesan ke semua client, join dan leave notifications untuk memberikan informasi status client, error handling untuk menangani koneksi yang terputus, serta dual threading pada client untuk komunikasi non-blocking. Aplikasi ini cocok untuk implementasi chat room sederhana atau group chat dalam jaringan lokal atau internet.
